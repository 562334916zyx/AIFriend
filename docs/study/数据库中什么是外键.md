在 PyCharm 里说的**外键**，一般不是 Python 语法，而是**数据库里的概念**。

**外键（Foreign Key）**就是：一个表中的字段，用来关联另一个表的主键。

比如有两个表：

### 学生表 `student`

| id | name |
| -- | ---- |
| 1  | 张三   |
| 2  | 李四   |

这里 `id` 是学生表的**主键**。

### 成绩表 `score`

| id | student_id | subject | score |
| -- | ---------- | ------- | ----- |
| 1  | 1          | 数学      | 90    |
| 2  | 2          | 英语      | 85    |

这里的 `student_id` 就可以作为**外键**，它关联的是 `student` 表里的 `id`。

意思是：

```text
score.student_id  →  student.id
```

也就是说，成绩表里的每条成绩，都属于某个学生。

### 外键的作用

外键主要用来保证数据关系正确。

比如 `student` 表里只有学生 `1` 和 `2`，那 `score.student_id` 就不应该随便写成 `999`，因为根本没有这个学生。

### SQL 示例

```sql
CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);

CREATE TABLE score (
    id INT PRIMARY KEY,
    student_id INT,
    subject VARCHAR(50),
    score INT,
    FOREIGN KEY (student_id) REFERENCES student(id)
);
```

这句就是定义外键：

```sql
FOREIGN KEY (student_id) REFERENCES student(id)
```

### 简单理解

**主键**：用来唯一标识自己表里的一条数据。
**外键**：用来连接另一个表的数据。

在 PyCharm 中，如果你连接了数据库，看到某个字段标记为外键，意思就是这个字段和另一个表有关联。
